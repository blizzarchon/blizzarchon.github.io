Preface

The modding tool described herein is only useful in the context of the data files, so if you are unfamiliar with making changes to data files it will be harder to understand the usage of this tool.

If you are unfamiliar, to get your bearings, I encourage you to look through the game files, which you can do from Slipstream Mod Manager via File → Extract Dats... Also see a modding tutorial about data files such as this one (see parts 1 and 5).

What I'm about to explain is listed in the readme_modders.txt file which can be found wherever you installed Slipstream, under Advanced XML, however, there aren't any usage examples which is one of the things I will remedy in this guide. You will not need to have read that section in order to understand this; I will be explaining it all here.

Motivations

The main idea behind Slipstream tags is to modify specific segments of FTL code and change only those segments.

Terminology

It's important to define some terms to make sure we're all on the same page here. Take a look at this event: 

			
<event name="MY_EVENT">
	<text>There seems to be nothing at all at this beacon. Time to move on... except, what's that over there?</text>
	<choice>
		<text>Investigate.</text>
		<event>
			<text>There's nothing after all. Bummer.</text>
		</event>
	</choice>
</event>
  • Tag

    'event', 'text' and 'choice' are tags. You can tell as much by the angled brackets '<' and '>'. <event ...> is concluded by the closing tag </event>, containing other tags within. Sometimes the tag can be self-closing like so: <event/>.

  • Value

    The content of a tag. The value of the 'choice' tag on line 3 is: Investigate.

  • Attribute

    name="MY_EVENT" is an attribute - a key-value pair. In this case, 'name' is the key, 'MY_EVENT' the value. The same tag cannot have more than one of the same key, as the keys are unique.

  • Children

    These are the tags directly inside of a tag. For instance, the first 'text' and 'choice' tags are the children of the 'event' tag with name="MY_EVENT".
    Therefore the relationship between the 'choice' on line 3 and the 'text' on line 4 is a parent-child relationship. It may sound a little odd but it will be useful to view the tags in this way.
